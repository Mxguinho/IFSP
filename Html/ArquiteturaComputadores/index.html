Entendendo o DMA (Direct Memory Access)

Tradicionalmente, a CPU é a mediadora de todas as transferências de dados no computador. Se você precisa ler um arquivo do disco para a RAM, a CPU precisa copiar byte por byte. Isso desperdiça poder de processamento precioso.

Para resolver isso, criaram o DMA (Acesso Direto à Memória). Um subsistema que permite que periféricos (Placas de vídeo, Rede, Discos, e portas Thunderbolt/USB4) conversem diretamente com a Memória RAM, contornando completamente a CPU.

O Problema de Segurança: Se um dispositivo tem acesso direto e irrestrito à RAM, o que impede um dispositivo malicioso (como um pendrive Thunderbolt) de ler suas senhas, chaves de criptografia ou injetar código direto na memória do Sistema Operacional?

A Lição

O DMA é essencial para a performance e eficiência energética de sistemas modernos. Sem ele, sua CPU ficaria estrangulada (100% de uso) copiando bytes sempre que você abrisse um arquivo pesado ou jogasse algo, deixando o sistema inteiro lento.

A Lição (O Ataque)

A "liberdade" de acesso total à RAM significa que o Sistema Operacional e os Antivírus não percebem o ataque. O hardware (placa-mãe) apenas atende ao pedido elétrico de leitura feito pelo barramento PCIe/Thunderbolt. Senhas de administrador, chaves do BitLocker e dados sensíveis ficam expostos na memória física.

O Porteiro IOMMU (A Defesa)

Como o hardware moderno se defende: Input-Output Memory Management Unit.

Você é o controlador do IOMMU. Sua função é traduzir e restringir quais partes da RAM cada dispositivo PCIe pode acessar. Atribua as permissões corretas (arraste as "Fichas" para os dispositivos) para evitar que o PC seja invadido por periféricos não confiáveis, mas mantenha a Placa de Vídeo rápida!

A Lição (A Defesa)

O IOMMU (VT-d na Intel, AMD-Vi na AMD) funciona como um MMU para periféricos. Ele cria tabelas de página que interceptam pedidos de DMA. Se um dispositivo tenta ler um endereço para o qual não tem mapeamento válido, o hardware corta o acesso e dispara uma falha no sistema. Isso impede que portas externas (Thunderbolt/Firewire) roubem dados, exigindo que o SO autorize a memória explicitamente (Sandboxing).

O Simulador de "Evil Maid"

Cenário: Você deixou seu notebook bloqueado (Win+L) no quarto do hotel e desceu para jantar. A camareira maliciosa entra.

1. A Conexão

A atacante conecta um dispositivo customizado (como um PCIe Screamer ou placa via Thunderbolt) na porta lateral do seu notebook ligado.

2. Disparo do Script

O hardware malicioso se passa por uma placa de rede confiável e emite pedidos de TLP (Transaction Layer Packet) pela rede PCIe para ler a RAM.

3. Localizar 'lsass.exe'

O script varre a memória em busca da assinatura do processo do Windows responsável pelo gerenciamento de credenciais na RAM.

4. Extrair Hash

O hash da sua senha é extraído via DMA para o pendrive em menos de 10 segundos. O invasor desconecta e vai embora. A tela de bloqueio nunca mudou.

Notebook Comprometido!

A Lição (Segurança Física)

O bloqueio de tela via software (Windows Lock) protege contra o uso do teclado e mouse. Ele não protege contra ataques de barramento em nível de hardware. Para evitar "Evil Maid Attacks" via DMA, os sistemas operacionais modernos adotam o "Kernel DMA Protection", utilizando o IOMMU para bloquear periféricos externos enquanto a tela estiver bloqueada.
